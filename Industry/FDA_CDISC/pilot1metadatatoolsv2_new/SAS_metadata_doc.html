This data standard describes the standard metadata sets.  These metadata sets can
describe the structure, attributes and allowed values of any SAS library (including
data sets, views, and format catalogs).  In fact, the metadata is used here to
describe the metadata structures and generate the information you are reading now,
since the metadata is a SAS library.
 
The metadata names use relation database terms instead of SAS terms because the
metadata can describe other databases, such as oracle schemas and tables.  Thus,
for example, the term "table" is used and is synonymous with "SAS data set" and
the term "column" is used and is synonymous with "SAS variable".  A number of SAS
macros have been created that can process the information in the metadata sets in
a way that is not possible when data requirements are entered into word documents.
For example, the mdatribs macro generates all the label and format statements when
building the data library being described - the information is entered one time
and used many times during the process of creating the database being described.
Other macros also use the metadata to assist during the requirements, build and
validation of the database.
 
The metadata is comprised of
 
        TABLES          Describes attributes of a table
        COLUMNS         Describes the attributes of each column in a table
        COLUMNS_PARAM   Describes the attributes of each value that a
                         parameter column in a tall-thin data set has
        VALUES          Describes the valid values and SAS user formats
                         that a column or column-parameter can have
        DESCRIPTIONS    This is a SAS catalog that allows entry of any
                         text to describe the derivation logic or any
                         other information about a column
 
TABLES has a one to many relationship to COLUMNS, joined by the primary key column
named "table".
 
COLUMNS has a one to many relationship to COLUMNS_PARAM, joined by the primary key
columns named "table" and "column".
 
COLUMNS_PARAM is the same structure as COLUMNS except it has one additional primary
key called "PARAM" and it does not have the CPKEY column.  The PARAM variable allows
specification of the values of the parameter column in a tall-thin data set.  These
parameter variables are associated with a value column in the tall-thin data set,
such that the value variable contains the value or result of the measurement
named in the parameter variable.  The COLUMNS_PARAM metadata set allows a full
description of the attributes of the parameter and value variables as if they were
a column in a wide data set.  The names of the variables in COLUMNS_PARAM are the
same as the variables in COLUMNS, except the non-key variables begin with "P"
instead of "C".
 
VALUES allows specification of the valid vallues that a COLUMN or a COLUMN_PARAM
can have.  Thus, there is a one to many relationship from COLUMNS to VALUES and
a one to many relationship from COLUMNS_PARAM to VALUES.  COLUMNS joins to
VALUES by the foreign key CFORMAT in COLUMNS to FMTNAME in VALUES.  COLUMNS_PARAM
joins to VALUES by the foreign key PFORMAT in COLUMNS_PARAM to FMTNAME in VALUES.
 
DECRIPTIONS is a catalog that contains source entries that allow any text to be
typed into the metadata to describe a table, a column or a parameter column.  It
can also hold text that contains notes about the library or schema being
specified (the note you are now reading is a specification-level note).  The
TABLE is linked to is description via the TDESCRIBE variable in the TABLES
metadata set - the TDESCRIBE variable contains the name of the entry.  A column
is linked to its description by either entering the name of the entry in the
CDESCRIPTION variable in the COLUMNS metadata set or by simply creating an
entry whose name is the same name as the COLUMN.  A parameter column is linked
to its entry in the same way as the column is linked - there is a PDESCRIPTION
variable in the COLUMNS_PARAM metadata set.  Note that one entry may be linked
to one or more columns or column parameters by linking multiple columns to
one entry.  This is useful, for example, to describe a primary key column that
is common to multiple data sets when the description of that column is the same
in each of the data sets.

 Data Set:  TABLES  Column:  TORDER 
General Description:
 
    The TORDER variable allows you to define the order that the tables
    will be displayed in when printing the data requirements residing
    in the metadata.  The mdprint macro will print the TABLE name in
    the order defined by TORDER.  If no TORDER is specified for any
    table, the table name will be listed in alphabetic order.  The
    order is defined by the BY statement TORDER TABLE, so if you
    specify TORDER for some tables but not all tables then the table
    names will print the unordered tables alphabetically followed
    by the table it TORDER order.

 Data Set:  COLUMNS  Column:  CFORMAT 
General Description
 
   Name of the format to associate with COLUMN or PARAM.  This can be:
 
     1 A SAS system format name; e.g. date9 or 6.2
 
     2 A user defined format where the valid values are defined in the START
       variable of the VALUES metadata set.  The FLABEL and FLABELLONG variables
       in the VALUES data set are optional - if there are decodes for START then
       these can be entered into the label variables, but if there are not decodes
       of START then the label variables can be blank.  When the label variables
       are blank, a user format is not defined, instead a simple list of valid
       values exists in the START variable of the VALUES metadata set.
 
     3 A user defined format where the valid values are defined in the FLABEL
       variable of the VALUES metadata set
 
     4 A user defined format where the valid values are defined in the FLABELLONG
       variable of the VALUES metadata set
 
   The CFORMATFLAG in COLUMNS and the PFORMATFLAG in COLUMNS_PARAM
   specifies which type of CFORMAT or PFORMAT is defined, respectively.
 
   Do not include trailing dots in CFORMAT.  Do not end format names with a number,
   since this is not allowed by SAS.  Character variables do not need formats of
   the type $w.

 Data Set:  COLUMNS  Column:  CHEADER 
General Description
 
   A flag to indicate whether the current COLUMN is a header variable or not.
 
   If a variable is a header variable then that variable will be added to
   all the other data sets that include the same primary keys (i.e. where
   CPKEY is true).  Thus, there is no need to re-enter header variables
   in all the other data sets.  The COLUMN should be flagged as a header
   variable with CHEADER in the source data set and it will be automatically
   copied to the other target data sets by the md macros.  For example,
   you can set CHEADER to "1" in a demographics data set for the COLUMN
   AGE and the AGE column will be automatically added to all other data
   sets that include all the same primary keys as the demographics data
   set.

 Data Set:  COLUMNS_PARAM  Column:  PARAM 
General Description
 
   A tall-thin data set contains at least one primary key variable that contains
   the name of the parameter being measured.  The name of this variable is
   specified in the COLUMNS metadata set and the values this variable can have
   are defined in the COLUMNS_PARAM metadata set.  All of the attributes of a
   variable must also be specified for each value of the parameter variable.  The
   tall-thin data set also contains a column that contains the value the
   parameter has.  For example, a parameter variable called VSNAME may contain
   values like PULSE and BYSYS (for pulse rate and systolic blood pressure).  A
   second variable contains the values of PULSE and BYSYS.  This second variable
   is numeric and has only one format, label, etc. associated with it.  The
   COLUMNS_PARAM data set allows specification of the format, label, etc. of BPSYS
   and of PULSE so the one values variable can have different attributes that
   depend on the value of the parameter variable.

back to data set COLUMNS_PARAM

 Data Set:  COLUMNS_PARAM  Column:  PARAMREL 
General Description
 
   The PARAM related variable name.  This is the name of a varaible in a
   tall-thin data set whose attributes change according to the value of
   the PARAM variable.  All PARAM variables will have a numeric or
   character result variable, for example, and the attributes of that one
   variable in the tall-thin data set should be different for each value
   of the PARAM variable.  For example, the format of a blood pressure
   (DIA) result is different than the format of a body fat pecentage (BFP)
   but both the result values are stored in the same variable in a
   tall-thin data set.  The PARAMREL allows you to select variables from
   COLUMNS in a tall-thin data set and further define them for each
   different PARAM value.

back to data set COLUMNS_PARAM

 Data Set:  COLUMNS_PARAM  Column:  PARAMRELCOL 
General Description:
 
 
    PARAMRELCOL allows you to assign a name to the combination of
    PARAM and PARAMREL.  This name can be applied when creating a
    short-wide data set from a tall-thin data set.  That PARAMREL
    variable maps to several variables in a short-wide data set -
    one variable be PARAM value.  The name of this variable in the
    short-wide data set can be defined in the PARAMRELCOL meta
    variable.
