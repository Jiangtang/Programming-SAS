SPECTRE (CLINICAL) RELEASE LEVEL
=================================

Read about the next planned release here.

RELEASE 2.2 - 25 MAY 2007

Changed %popfmt to allow the dropping of round brackets from the (N=xxx) population labels

Changed %unistats and %npcttab so that the percent sign can be forced to appear

Changed %unistats and %npcttab so that the percent format can be specified
 

RELEASE 2.1 - 24 MARCH 2007       (THE WINDOWS UPDATE RELEASE)

Changed %unistats to use "A0"x instead of "FE"x for decimal point alignment so that the "þ" characters no longer appear

Two part page number labels such as "Seite x" are allowed in the %pagexofy macro

Added handling in %unistats for the S.E.M. (standard error of the mean)

"nocenter" option support added for %unistats and %npcttab

Allows for more than a three digit "n" for %unistats category counts
 

RELEASE 2.0 - 10 MARCH 2007       (THE WINDOWS RELEASE)

Made Windows compliant using Cygwin but PostScript and PDF creation not functional for Windows.

Made easier to install by minimizing macro and script edits.

Instructions for the Windows version installation added.

All macros put out their version number on first invocation so reruns done at a later date can be done using functionally identical macros.

The "protocol" and "titles" dataset libref location is now indicated by the global macro variable _ptlibref_ set up in the allocation macros and all scripts and macros now refer to this variable to locate these datasets.

Document added to show Spectre reporting from empty directories all the way through to final report run plus show how Spectre can be a "push-button" reporting system.

New global macro variable _popdone_ added to indicate to reporting macros whether or not the population label has already been included in the titles by the client titles macro.

%pagexofy macro written to replace the "pagexofy" script and allow more flexibility for the page label style.

%npcttab v5.0 allows user-formatted level variables.

%npcttab v6.0 adds the dsall= parameter so that it shows all categories in the dataset optionally assigned to it with zero counts or blanks if not in the input data.
