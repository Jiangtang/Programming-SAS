Spectre (Clinical) next Release

(Author: Roland Rashleigh-Berry                                                                                Date: 03 Oct 2007)

Introduction

Releases tend to be "rolling releases" with updates available when ready with no strict release date unless it is helpful as a cutoff for stability purposes. The readiness of a new module will be indicated by the text "DONE" in the list below.

Release 2.3  --  December 2007

  • Allow for variable labels to be attached to variable names in the %unistats call and add a switch so that numeric processing can be performed on numeric variables to override any user-defined format attached. (DONE)
  • Change %crprotds, %protinfo and %closerep so that the user can specify the %pagexofy macro to be used in protocol.txt to allow for customized handling of page labels plus other needed editing (DONE)
  • Allow for "extra analysis" programs to run after the "final run" so that their titles are stored in the titles dataset and they can be run as a suite and bookmarked PDFs can be created from their output in the same way as is done for normal tables and listings. (DONE)
  • Allow users to override the derived "lis file" names using a new "lisfile:" line in the .titles member and to alert users to lis file name remappings in the crtitlesds messages (DONE)
  • Change script "alltitles" so that it will accept when there are no .titles members and assume ALL.TITLES has been created by a different utility (such as spreadsheet extraction) (DONE)
  • Change %unistats and %npcttab so by default they put out warning messages when they encounter a percentage value > 100.01 (DONE)
  • Allow saving the "proc report" output dataset from %unistats and %npcttab (DONE)
  • Add facility to %unistats to create cell tables suitable for inclusion and conversion to a table in a word processor (DONE)
  • Add new style for output from %unistats to create a column report rather than an indented report (DONE)
  • Add an ODS statement pass-through facility to %unistats and %npcttab (DONE)
  • Add a titles and footnotes macro name facility to %unistats to activate titles and footnotes after p-values have been created so that they can be used in the titles and footnotes (DONE)
  • Enhance the %pagexofy macro so that for page target characters placed one character before the end of the line it will attempt to recenter the line after adding the "(Page x of Y)" label.
  • Add appropriate error checking of "lis file" output file name overrides
  • Enhance the %pagexofy macro so that it can optionally add up to 5 "pre title" lines and 5 "post footnote" lines to the final output so that the 10 titles and 10 footnotes limit can be broken (allowing the programmer to use all 10 titles and footnotes as user-defined) and allow for the "Page x of Y" target character "FF"x to be present in one of these lines. This macro can then be called from the client %pagexofy macro defined in protocol.txt .

Conclusion

The changes in the next release are listed above. Those changes not yet implemented are subject to change until the last day.
