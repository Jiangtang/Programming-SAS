Connect to Oracle and read System Tables

Author: Roland Rashleigh-Berry
Updated: 12 Mar 2013
You will sometimes want to connect to Oracle and read the system tables such as the very useful table DBA_SCHEDULER_JOB_RUN_DETAILS which has details on scheduled job runs. You can do this using a very simple connection to Oracle like this. Add your own user, password and path. The "put" statement should be used as you see it as it tells you about the status of the sql connection:
 
proc sql noprint;
CONNECT to oracle
(user='*******' password='******' path='****************');
%put &sqlxmsg;
create table job_logs as SELECT *
FROM connection to oracle
(SELECT * from DBA_SCHEDULER_JOB_RUN_DETAILS) ;
%put &sqlxmsg;
DISCONNECT from oracle;
quit;
